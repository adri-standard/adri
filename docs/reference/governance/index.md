## Charter

**Document**: [Charter →](charter.md)

The ADRI Charter defines our fundamental mission, vision, and values as an open standard for AI agent-data interoperability.

### Key Elements

**Mission**: Establish and maintain an open standard that enables seamless interoperability between AI agents and data sources, accelerating the adoption of reliable AI systems across all industries.

**Vision**: A world where any AI agent can work with any data source through a common standard, unlocking exponential innovation and making 99% reliable AI the norm.

**Core Values**:
- **Openness**: Standards developed in public with all voices welcome
- **Interoperability**: Break down data silos and enable ecosystem innovation
- **Practicality**: Solve real problems with immediate value
- **Quality**: Technical excellence and comprehensive testing
- **Community**: Collective ownership and collaborative development

**Strategic Goals**:
- **Year 1 (2025)**: Foundation - governance, v1.0 spec, reference implementations
- **Year 2 (2026)**: Adoption - 100+ organizations, certification program
- **Year 3 (2027)**: Ecosystem - 1000+ certified sources, marketplace operational

## Governance

**Document**: [Governance →](governance.md)

The ADRI Governance document outlines how decisions are made, who makes them, and the processes for community participation.

### Governance Structure

#### 1. Community Contributors
- **Who**: Anyone interested in improving ADRI
- **Role**: Submit issues, propose changes, participate in discussions
- **Rights**: Vote on major decisions through GitHub discussions

#### 2. Maintainers
- **Who**: Active contributors elected by the community
- **Role**: Review PRs, guide technical direction, ensure quality
- **Term**: 1 year, renewable

#### 3. Steering Committee
- **Who**: 5-7 members representing diverse stakeholders
- **Role**: Strategic direction, conflict resolution, charter updates
- **Composition**: Technical maintainers, enterprise users, vendors, academia
- **Term**: 2 years, staggered

#### 4. Working Groups
- **Template Working Group**: Industry-specific standards
- **Implementation Working Group**: Reference implementations
- **Ecosystem Working Group**: Marketplace and partnerships

### Decision Making Process

#### Minor Changes (Bug fixes, clarifications)
1. Pull request submitted
2. Maintainer review (1 approval needed)
3. 48-hour comment period
4. Merge if no objections

#### Major Changes (New features, breaking changes)
1. Proposal created as GitHub issue
2. 2-week RFC (Request for Comments) period
3. Working group recommendation
4. Steering committee vote (simple majority)
5. 1-week final comment period
6. Implementation

## Code of Conduct

**Document**: [Code of Conduct →](code-of-conduct.md)

All ADRI community participants must follow our Code of Conduct, which is based on the Contributor Covenant v2.0.

### Our Pledge

We pledge to make participation in our community a harassment-free experience for everyone, regardless of age, body size, visible or invisible disability, ethnicity, sex characteristics, gender identity and expression, level of experience, education, socio-economic status, nationality, personal appearance, race, religion, or sexual identity and orientation.

### Standards

**Positive Behavior**:
- Demonstrating empathy and kindness toward other people
- Being respectful of differing opinions, viewpoints, and experiences
- Giving and gracefully accepting constructive feedback
- Accepting responsibility and apologizing for mistakes
- Focusing on what is best for the overall community

**Unacceptable Behavior**:
- Sexualized language or imagery, and sexual attention or advances
- Trolling, insulting or derogatory comments, and personal or political attacks
- Public or private harassment
- Publishing others' private information without permission
- Other conduct inappropriate in a professional setting

### Enforcement

Violations can be reported through:
- Private security advisory on GitHub
- Direct contact with maintainers
- Community moderation channels

Enforcement follows graduated consequences from correction to permanent ban based on severity and pattern of behavior.

## Decision History

### Major Decisions

#### 2025 Decisions
- **January**: Charter ratification and governance structure establishment
- **February**: Reference implementation language selection (Python)
- **March**: Template specification format (YAML-based)
- **April**: Assessment mode framework (Discovery vs Validation)
- **May**: Dimension scoring methodology finalization
- **June**: Documentation architecture restructuring

### RFC Process

All major changes follow the RFC (Request for Comments) process:

1. **Proposal**: Submit detailed RFC as GitHub issue
2. **Discussion**: 2-week community comment period
3. **Revision**: Incorporate feedback and update proposal
4. **Working Group Review**: Technical evaluation and recommendation
5. **Steering Committee Vote**: Final decision with rationale
6. **Implementation**: Development and testing phase
7. **Release**: Integration into next version

### Voting Records

Steering committee votes are recorded with:
- **Proposal summary**: What was being decided
- **Vote tally**: For/against/abstain counts
- **Rationale**: Reasoning behind the decision
- **Dissenting opinions**: Minority viewpoints when applicable

## Amendment Process

### Governance Changes

This governance framework can be amended through:

1. **Proposal**: Submit amendment as GitHub issue with detailed rationale
2. **Community Input**: 30-day comment period for feedback
3. **Working Group Review**: Governance working group evaluation
4. **Steering Committee Vote**: 2/3 supermajority required for approval
5. **Community Ratification**: Simple majority vote in community poll
6. **Implementation**: 30-day notice period before changes take effect

### Charter Changes

Charter amendments require:
- **60-day comment period**: Extended discussion for fundamental changes
- **Steering Committee Approval**: 2/3 supermajority vote
- **Community Ratification**: Simple majority in community-wide vote
- **Documentation Update**: All references updated within 30 days
